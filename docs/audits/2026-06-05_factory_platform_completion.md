| Stale Sentry issues | ⏳ | Requires manual review and grouping |

### Stable-Branch Pattern Details

**Before:** Each workflow run created timestamped branch (e.g., `chore/cost-snapshot-2026-06-05-1715`)
- Result: Multiple duplicate PRs per day if workflow runs > once

**After:** Each workflow run uses date-based branch (e.g., `chore/cost-snapshot-2026-06-05`)
- Check if branch exists via `git ls-remote`
- Reuse if found, update PR instead of creating new one
- Force-push with `--force-with-lease` for safety
