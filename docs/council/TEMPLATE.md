# Council Inquiry Template

## Metadata

| Field | Value |
|---|---|
| Inquiry ID | C-### |
| Title | Short, specific title |
| Author | Name or team |
| Date Opened | YYYY-MM-DD |
| Status | draft \| review \| approved \| deferred \| rejected \| superseded |
| Desired Decision Date | YYYY-MM-DD |
| Primary Owner | Team or person |
| Related Docs | Links |

## 1. Decision Needed

State exactly what the council is being asked to decide.

## 2. Problem

What problem or opportunity makes this inquiry necessary?

Include:

1. current state
2. pain or risk
3. why now

## 3. Recommended Path

Describe the recommended path in direct terms.

Include:

1. what should be done
2. what should not be done
3. why this is the best current path

## 4. Alternatives Considered

For each alternative, include:

1. summary
2. pros
3. cons
4. why it was not recommended

## 5. Impact

Cover the impact on:

1. platform
2. Studio UX
3. delivery speed
4. governance and review load
5. long-term maintenance

## 6. Approval Criteria

What conditions must be true for this inquiry to be approved?

## 7. Risks and Mitigations

List the highest-risk failure modes and how they will be handled.

## 8. Council Questions

Make the open questions explicit and answerable.

## 9. If Approved, Next Actions

List the first concrete actions, with owners if known.

## 10. Outcome

Fill this in once the council decides.

### Decision

Approved, deferred, rejected, or superseded.

### Notes

Summarize the decision and any conditions.
