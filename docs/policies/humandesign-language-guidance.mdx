# HumanDesign Language Guidance

## Purpose

This policy defines canonical language for HumanDesign (selfprime) product surfaces.

Use this file as the source of truth for copy review, UI text updates, and compliance checks.

## Canonical Terms

- Use **Energy Blueprint** for product terminology.
- Use **Energy Chart** for product terminology.

## Field-Term Exception (SEO Guides)

Field terms such as "Human Design" and "Bodygraph" are allowed only in approved SEO/education surfaces where discoverability requires it.

Examples of allowed exception surfaces:

- Public guides and educational content hubs.
- Structured long-form content explicitly marked as educational/reference.

## Product-Surface Restrictions

Outside approved SEO exceptions, do not use field terms in:

- App UI labels and navigation.
- Marketing hero copy and metadata for core product pages.
- In-app empty states, helper text, and transactional flows.
